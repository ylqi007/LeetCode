## Analysis
* `sum = nums[i] + nums[j] + nums[k]`, and `diff = target - sum`, which could be positive or negative.
* The difference should use `Math.abs()` to represent.

> `diff = target - sum` ==> `sum = target - diff`

```java
class Solution {
    public int threeSumClosest(int[] nums, int target) {
        int diff = Integer.MAX_VALUE;
        int n = nums.length;
        
        Arrays.sort(nums);		// sort
        for(int i=0; i<n && diff!=0; i++) { // could be nums.length-2; diff==0æ—¶ï¼Œåœæ­¢å¾ªçŽ¯
            int lo = i + 1;
            int hi = n - 1;
            while(lo < hi) {
                int sum = nums[i] + nums[lo] + nums[hi];
                if(Math.abs(target - sum) < Math.abs(diff)) {
                    diff = target - sum;	// sum = target - diff
                }
                if(sum < target) {
                    lo++;
                } else {    // must be "else" here, or the while-loop will stuck and cause TLE
                    hi--;
                }
            }
        }
        return target - diff;
    }
}
```
**Notes:**
* :warning:æ³¨æ„ When `diff == 0`, then we found the minimum difference, i.e. 0.

* Time Complexity: `O(n^2)`. We have outer and inner loops, each going through n elements. Sorting the array takes `O(nlogâ¡n)`, so overall complexity is `O(nlogâ¡n+n^2)`. This is asymptotically equivalent to `O(n^2)` (è¿™æ¸è¿‘ç­‰ä»·äºŽ`O(n^2)`).
* Space Complexity: from `O(logâ¡n)` to `O(n)`, depending on the implementation of the sorting algorithm.


## Method 2. Binary Search
> `diff = complement - nums[hi] = target - nums[i] - nums[j] - nums[hi]`, i.e. `sum = nums[i] + nums[j] + nums[hi] = target - diff`.
>
> or
>
> `diff = complement - nums[lo] = target - nums[i] - nums[j] - nums[lo]`, i.e. `sum = nums[i] + nums[j] + nums[lo] = target - diff`

```java 
class Solution {
    public int threeSumClosest(int[] nums, int target) {
        int diff = Integer.MAX_VALUE;
        int n = nums.length;
        Arrays.sort(nums);
        for(int i=0; i<n && diff!=0; i++) {
            for(int j=i+1; j<n-1; j++) {
                int complement = target - nums[i] - nums[j];
                int idx = Arrays.binarySearch(nums, j+1, n-1, complement);
                int hi = (idx >= 0) ? idx : ~idx;
                int lo = hi - 1;
                if(hi < n && Math.abs(complement - nums[hi]) < Math.abs(diff)) {
                    diff = complement - nums[hi];
                }
                if(lo > j && Math.abs(complement - nums[lo]) < Math.abs(diff)) {
                    diff = complement - nums[lo];
                }
            }
        }
        return target - diff;
    }
}
```

**Complexity Analysis:**

- Time Complexity: `O(n^2 logâ¡n)`. Binary search takes `O(logâ¡n)`, and we do it `n` times in the inner loop. Since we are going through `n` elements in the outer loop, the overall complexity is `O(n^2 logâ¡n)`.
- Space Complexity: from `O(logâ¡n)` to `O(n)`, depending on the implementation of the sorting algorithm.
